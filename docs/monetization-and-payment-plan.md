# 收款与变现路线

## 当前阶段：先工具，再文章

第一阶段的核心顺序是：

1. 先把工具做成可用入口。
2. 用文章解决长尾搜索问题。
3. 每篇文章都导流到相关工具、模板或服务 CTA。
4. 通过真实使用数据判断哪些主题值得继续扩展。

原因很简单：文章可以带来搜索流量，但工具才能让用户留下来、复制结果、下载模板、联系服务。先发大量文章而工具体验很弱，会浪费早期流量。

## 第一阶段变现：服务引流

最适合当前项目的收款方式不是立刻做会员，而是从轻服务开始：

- Codex / Claude Code / GitHub / Vercel 一对一配置。
- 报错排查和部署陪跑。
- Upwork Proposal 人工润色。
- 新手作品集页面搭建。
- 小网站交付流程指导。

建议先在联系页放清楚服务范围和邮箱，不要承诺结果，不要承诺接单成功。

## 第二阶段变现：模板包

当模板下载页有稳定访问后，可以把免费模板和付费模板分开：

- 免费：Proposal 基础模板、客户问题清单、交付检查清单。
- 付费：完整接单 SOP、报价表、复盘表、作品集文案包、部署检查包。

可选支付工具：

- Gumroad：适合快速卖数字模板，配置简单。
- Lemon Squeezy：适合数字产品和国际支付。
- Paddle：适合更正式的软件/数字产品税务处理。
- Stripe：灵活但配置和合规成本更高。

第一版建议优先 Gumroad 或 Lemon Squeezy，少写代码，先验证需求。

## 第三阶段变现：联盟链接

工具导航可以逐步接入联盟链接：

- 域名和主机：Namecheap、Cloudflare 等。
- 自动化工具：Make、Zapier。
- 设计工具：Canva。
- 数字产品支付：Gumroad、Lemon Squeezy。

必须明确展示 affiliate disclosure，不能把广告伪装成客观排名。工具推荐要写适合谁、不适合谁、风险和替代方案。

## 第四阶段变现：广告

当搜索流量稳定后再接广告：

- Google AdSense
- Carbon Ads 或开发者工具类广告
- 自营广告位

广告位应该放在文章中后段、文章底部或工具页边栏，不要影响工具使用。

## 收款路径建议

### 海外服务收款

如果服务来自 Upwork / Fiverr，优先使用平台内收款，遵守平台规则，不要引导客户站外付款。

### 自有网站服务收款

早期可以先人工沟通后使用：

- Wise
- Payoneer
- PayPal

注意事项：

- 写清服务范围。
- 写清是否退款。
- 写清交付物。
- 不承诺收入结果。
- 保留沟通记录和发票/收据。

### 数字模板收款

建议使用：

1. Gumroad 或 Lemon Squeezy 上架模板。
2. 模板页按钮跳转到购买链接。
3. 购买后由平台自动交付文件。
4. 网站只做介绍、预览和 SEO，不直接处理支付。

## 不建议现在做的事

- 不建议立刻做会员系统。
- 不建议自己接 Stripe Checkout 和 webhook。
- 不建议承诺陪跑后一定接到单。
- 不建议卖“保证赚钱”的课程。
- 不建议把未审核 AI 文章批量发布。

## 推荐执行顺序

1. 打磨 Proposal 生成器、报错解释器、报价助手。
2. 完善模板下载页，先放免费模板。
3. 增加联系页服务说明。
4. 发布第一批 3-5 篇高质量文章。
5. 接 Google Search Console。
6. 根据搜索词继续生成草稿。
7. 有下载和咨询后再做付费模板。
