# AI 工具学习每日执行表检查清单

先给结论：每日执行清单要帮助新手每天完成一个可验证动作，而不是制造更多待办。检查重点是任务是否具体、输入是否齐全、工具是否固定、结果是否验证、风险是否可控、明天是否有下一步。

这份清单适合每天开始前和结束前各看一次。开始前用它选任务，结束前用它确认是否真的留下了证据。

## 适合谁

适合正在执行 AI 工具实践 30 天计划的新手。你可能每天时间不多，所以更需要把任务拆小。

也适合容易分心的人。清单可以帮你判断：今天是不是又换工具了，是不是又看教程但没有输出，是不是又跳过了验证。

## 不适合谁

不适合想用清单代替练习的人。填表只是辅助，真正重要的是每天完成一个小结果。

也不适合高风险客户项目。支付、生产数据库、安全漏洞、隐私数据和高权限账号，不应该被普通每日清单推动到交付。

## 具体步骤

每天按这个顺序检查：

1. 选一个唯一主任务。
2. 确认输入材料。
3. 固定今天使用的工具。
4. 写清验证方式。
5. 做风险检查。
6. 保存结果证据。
7. 写一句复盘和明天动作。

如果第 1 项不够具体，今天很可能会空转。

## 任务检查

```text
今日任务：
1. 是否只有一个主任务。
2. 是否能在 30-90 分钟内推进。
3. 是否能产生截图、链接、命令输出或文字记录。
4. 是否和作品集、草稿审核、项目判断或 Proposal 练习有关。
5. 是否不是“学完所有工具”这种大目标。
```

任务越小，越容易坚持。

## 输入检查

```text
输入材料：
1. 是否有明确练习题或客户需求。
2. 是否有截图、报错、链接或参考资料。
3. 是否知道今天要修改或整理什么。
4. 是否没有输入真实密钥、隐私数据或高权限账号。
5. 如果材料不齐，是否先把“补材料”作为今天任务。
```

材料不齐时，直接开做通常会浪费时间。

## 工具检查

```text
今日工具：
1. 只选一个 AI 工具做主要辅助。
2. 只选一个代码或文档工作区。
3. 不临时更换整个工具链。
4. 有必要时才查新教程。
5. 工具输出需要人工确认。
```

固定工具不是保守，是为了让执行有节奏。

## 验证检查

```text
验证方式：
1. 页面能打开。
2. 构建能通过。
3. 报错能解释。
4. 文案能读通。
5. 草稿质量检查通过。
6. 截图前后能对比。
7. 能写出未覆盖范围。
```

至少选择一种验证方式。没有验证，就不算完成。

## 风险检查

```text
风险边界：
1. 是否涉及客户真实数据。
2. 是否涉及生产环境。
3. 是否涉及支付、广告、收款或 API Key。
4. 是否涉及平台规则。
5. 是否会对外公开。
6. 是否需要人工复核。
```

命中风险项时，把今天任务改成“整理风险和确认问题”，不要继续对外交付。

## 结果检查

```text
今日结果：
1. 是否有链接、截图或文件路径。
2. 是否记录 AI 帮了哪一步。
3. 是否记录自己人工修改了什么。
4. 是否记录验证结果。
5. 是否写了一句复盘。
6. 是否写了明天唯一下一步。
```

没有结果记录，明天很容易从头再来。

## 常见错误

第一个错误是把每日清单写成长期愿望。今天只能执行今天能推进的一步。

第二个错误是每天都换主题。昨天草稿、今天投标、明天做后端，最后都没有成型。

第三个错误是没有收尾。每天结束时必须写明天下一步。

第四个错误是忽略 draft 状态。AI 生成的内容没有审核前，不要公开。

## 风险提醒

不要承诺收入结果，不要承诺一定成交，不要把每日计划包装成职业保证。任何客户回复、公开文章、真实交付、收款和生产配置都需要人工确认。

如果今天接触到敏感数据、高权限账号、生产数据库或平台外结算请求，先暂停，把问题写进风险记录。

## 总结

每日执行清单的价值，是让你每天都留下一个真实结果。任务具体、输入清楚、工具固定、验证明确、风险可控、记录完整，才算完成一天。长期看，这比偶尔冲刺更可靠。
