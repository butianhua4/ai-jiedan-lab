# Google Search Console 新站怎么用

新站可以准备使用 Google Search Console，但不一定要马上进入后台操作。对新手来说，先检查公开页面、sitemap、robots、canonical、标题描述和重要链接，通常比急着验证站点更稳。真正需要登录、验证、上传文件、改 DNS 或查看搜索数据的步骤，必须等站点所有权和授权确认后再做。

这篇文章讲的是“接入前准备”和“新站初步使用顺序”，不是搜索排名承诺。Search Console 能帮助观察收录和搜索表现，但不能保证页面一定被收录、排名提高或带来收入。

## 适合谁

适合刚上线个人网站、作品集、工具页、模板页或客户小站的新手。你想知道新站是否已经有基础 SEO 文件、是否可以提交 sitemap、是否需要 Search Console 后台数据。

也适合 AI 项目练习。你可以先对公开页面做不需要授权的检查，保存截图和结论，等客户或站点所有者确认后再进入验证步骤。

## 不适合谁

不适合没有站点所有权、没有客户授权、无法确认域名归属的情况。只要需要改 DNS、上传验证文件、添加 meta 标签或进入客户后台，就不能擅自操作。

也不适合复杂 SEO 诊断、排名恢复、惩罚处理、大型站点抓取预算、国际化站点或新闻站点。那些场景需要更多数据、经验和权限。

## 风险提醒

不要为了证明会用工具而随便验证别人的站点。Search Console 验证通常意味着你可以看到搜索数据、提交 sitemap、查看覆盖情况，属于有权限边界的操作。

不要承诺“提交 GSC 后就会被收录”。提交 sitemap 只是帮助搜索引擎发现页面，不代表一定收录、排名或产生访问。

## 具体步骤

按这个顺序做：

1. 先打开新站首页和重要页面，确认可访问。
2. 检查标题、描述、canonical 和 noindex 状态。
3. 打开 `/sitemap.xml` 和 `/robots.txt`。
4. 确认 sitemap 只包含应公开页面。
5. 确认 robots 没有误拦截重要页面。
6. 保存公开检查截图和问题清单。
7. 确认站点所有权和客户授权。
8. 再计划 Search Console 验证方式。
9. 把登录、验证、后台截图、提交 sitemap 写入明日待办。

前六步不需要登录后台，适合先做练习和初查。

## 先做公开检查

新站最基础的检查包括：

- 首页是否 200
- 重要页面是否能打开
- 标题是否唯一
- 描述是否清楚
- canonical 是否指向正确域名
- 草稿是否没有进入 sitemap
- robots 是否指向 sitemap
- 重要 CTA 是否能打开

这些检查可以用浏览器、命令行或站内脚本完成，不需要任何账号权限。

## 什么时候再进 GSC

满足这些条件后再考虑进入 Search Console：

- 站点已经公开
- 域名归属清楚
- 有站点所有者或客户确认
- 知道要验证哪个属性
- 知道选择哪种验证方式
- 有回滚或撤销方案
- 准备好记录验证过程
- 知道提交哪个 sitemap

如果只是“新站看起来应该装一下”，就先暂停。没有明确目的的后台操作，通常只会制造混乱。

## 常见验证方式要谨慎

常见验证可能涉及 DNS、HTML 文件、meta 标签或已有平台集成。每一种都可能影响站点或需要后台权限。新手不要在客户项目中自行选择方式。

更稳妥的话术是：

```text
Before setting up Google Search Console, I need to confirm site ownership, the preferred verification method, and whether I am allowed to edit DNS, upload files, or add verification tags. I will only proceed after these points are confirmed.
```

这段话把权限确认放在操作前面，适合客户沟通。

## 明日待办

以下事项先记录，不要临时操作：

- 登录 Google Search Console
- 确认站点所有权
- 选择 property 类型
- 修改 DNS
- 上传 HTML 验证文件
- 添加 meta 验证标签
- 提交 sitemap
- 查看覆盖报告
- 查看搜索查询和点击数据
- 给协作者分配权限

这些都需要真实账号或授权。没有确认前，文章和方案保持草稿。

## 新手可以先交付什么

在没有 GSC 权限前，你仍然可以交付一份安全的基础检查：

- 页面访问截图
- sitemap 和 robots 截图
- 是否出现草稿泄漏
- 重要页面列表
- 明显 404 或链接问题
- 标题和描述问题
- 待授权事项清单

这比擅自进后台更专业，也更容易被客户理解。
