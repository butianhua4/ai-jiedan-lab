# PayPal 和 Payoneer 哪个适合新手：新手检查清单

这份清单用于比较 PayPal 和 Payoneer 前的自查。完整说明可以看 [PayPal 和 Payoneer 哪个适合新手](/blog/paypal-vs-payoneer-beginner)。清单重点不是直接给出唯一答案，而是帮你判断当前订单更适合哪条已查证路径。

本文保持草稿状态，不替你注册账号、不替你绑定银行账户、不替你做法律、税务或会计判断。真实账户动作都应先进入明日待办。

## 适合谁

适合准备接海外小单、客户直接付款、平台提现或模板定制的新手。你可能还没有稳定收入，但希望先知道 PayPal 和 Payoneer 的比较顺序。

也适合已经有一个客户咨询的人。你可以用清单把订单来源、付款币种、客户地区、平台规则和交付证据先整理好。

## 不适合谁

不适合处理大额交易、公司财务、税务申报、复杂合同、敏感商品或高争议订单的人。这些事项需要专业复核。

也不适合想用收款工具替代业务流程的人。报价、合同、验收、退款和交付记录仍然要单独写清楚。

## 具体步骤

1. 先完成订单来源检查。
2. 再完成平台规则检查。
3. 分别完成 PayPal 和 Payoneer 检查。
4. 完成证据链和费用记录检查。
5. 把无法确认的问题写进明日人工复核清单。
6. 在查证前，不注册不必要账户，不向客户发送未确认付款路径。

## 1. 订单来源检查

- [ ] 订单来自自由职业平台。
- [ ] 订单来自客户直接付款。
- [ ] 订单来自模板、数字产品或下载页面。
- [ ] 订单还不存在，只是在提前准备。
- [ ] 我已经写下客户地区、付款币种、金额和交付内容。

如果订单还不存在，优先完善作品集、联系入口、报价说明和交付模板。收款工具可以晚一点再定。

## 2. 平台规则检查

- [ ] 我知道目标平台是否要求使用平台内置收款。
- [ ] 我知道平台是否支持 PayPal。
- [ ] 我知道平台是否支持 Payoneer。
- [ ] 我已经保存平台提现或收款规则链接，准备明日复核。
- [ ] 我不会为了方便跳过平台指定流程。

## 3. PayPal 检查

- [ ] 客户是否习惯或明确要求 PayPal，需要确认。
- [ ] 当前地区、账户类型、收款限制和提现路径，需要明日查证。
- [ ] 费用、汇率、退款和争议规则，需要明日查证。
- [ ] 服务范围、交付物、修改次数和验收方式，已经写清。
- [ ] 我不会用 PayPal 链接替代合同、发票或交付说明。

## 4. Payoneer 检查

- [ ] 目标平台是否支持 Payoneer，需要确认。
- [ ] 当前地区、账户资料、身份验证和银行绑定要求，需要明日查证。
- [ ] 费用、到账时间、提现路径和记录导出，需要明日查证。
- [ ] 姓名、地址、银行账户和平台资料是否一致，需要复核。
- [ ] 我不会因为别人推荐就马上上传敏感资料。

## 5. 证据链检查

- [ ] 付款前有明确报价和服务范围。
- [ ] 付款后能保存订单编号、付款状态和平台通知。
- [ ] 交付时能保存文件、链接、版本和客户确认。
- [ ] 退款和争议时能说明交付内容和验收过程。
- [ ] 每笔收入能对应到客户、项目、币种、费用和提现记录。

## 风险提醒

不要只比较“哪个便宜”。更关键的是平台是否允许、客户能否付款、你能否提现、证据是否完整、争议是否可处理。

不要在资料不清楚时注册。真实姓名、地址、银行账户、税务资料和平台资料如果不一致，可能带来审核或提现问题。

不要承诺收款一定顺利。账户限制、付款失败、退款、争议、汇率变化和平台规则变化都可能发生。

## 6. 明日人工复核清单

- [ ] PayPal 当前地区支持、账户类型、费用、提现、退款和争议规则。
- [ ] Payoneer 当前地区支持、账户资料、费用、提现路径和平台支持。
- [ ] 目标平台是否允许对应收款或提现方式。
- [ ] 是否需要税务资料、发票、合同、消费者提示或额外合规说明。
- [ ] 当前订单是否应暂缓收款，直到范围和证据链完整。
- [ ] 是否应该继续使用平台内置收款，而不是另开工具。
