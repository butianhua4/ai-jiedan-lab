# Make 和 Zapier 自动化新手检查清单

这份清单用于判断一个自动化需求是否适合用 Make 或 Zapier 处理。它不会替你登录账号、连接客户应用或验证真实数据，只帮你把风险点列出来：触发器、字段、权限、测试、日志、失败处理和维护边界。

自动化工具的界面很友好，但连接的往往是真实业务系统。新手最需要避免的不是“选错工具”，而是没确认权限、没测试数据、没失败方案，就把流程接到客户生产环境里。

## 适合谁

适合准备做表单到表格、订单提醒、Notion 记录、邮件归档、客服工单整理、简单 CRM 同步等低风险自动化的新手。你可以用这份清单判断需求是否边界清楚，是否能用假数据先跑通。

也适合准备用自动化项目写作品集的人。作品集里可以展示流程图、测试数据、日志截图和失败处理说明，但不应该展示客户敏感信息或真实联系人名单。

## 不适合谁

不适合直接处理支付、账号权限、生产数据库、隐私数据、员工绩效、医疗、法律、金融或大量用户通知的项目。即使 Make 和 Zapier 支持连接，也不代表新手可以独立交付。

也不适合客户只说“帮我把业务全自动化”的模糊需求。自动化必须拆到具体触发器和动作，否则报价、测试和验收都会失控。

## 检查一：触发器是否明确

- 触发来自表单提交、邮件到达、表格新增行、订单状态变化，还是人工点击？
- 触发频率大概是多少？
- 是否可能重复触发？
- 是否需要过滤条件？
- 触发失败时谁会知道？

如果触发器说不清，先不要选工具。先让客户提供一个真实但脱敏的例子，再画流程图。

## 检查二：数据字段是否可控

- 每一步读取哪些字段？
- 哪些字段会被改写、发送或同步到另一个工具？
- 是否包含姓名、邮箱、电话、订单、发票、付款、客户备注或内部标签？
- 是否有测试数据，还是只能用真实数据？
- 是否需要保留日志或导出记录？

字段越敏感，越要谨慎。新手练习可以从公开样例数据开始，客户项目则必须确认数据权限和保存周期。

## 检查三：权限和账号是否安全

- 连接的是你的账号、客户账号，还是团队空间？
- OAuth 授权会给工具哪些权限？
- 是否有只读权限或最小权限选择？
- 授权后谁可以编辑流程？
- 项目结束后如何移交或撤销访问？

需要登录、授权或查看后台的事项都先放入明日待办。不要在草稿里假装已经验证，也不要让客户随手发密码。

## 检查四：Make 和 Zapier 怎么取舍

- 如果流程线性、应用常见、客户需要快速理解，可以先评估 Zapier。
- 如果流程有分支、数据映射、循环、错误路径或多步转换，可以先评估 Make。
- 如果两者都要付费或都缺少关键连接，可能需要换工具或缩小范围。
- 如果流程影响真实业务，必须先做测试空间或样例数据 demo。

工具选择应该服务于可验证交付，而不是追求看起来更专业的界面。

## 具体步骤

1. 写下业务目标和验收标准。
2. 画出触发器、条件、动作和人工确认点。
3. 列出每一步需要读取、转换、写入或发送的数据字段。
4. 标记敏感数据和需要客户授权的账号。
5. 用样例数据分别评估 Make 和 Zapier 的可行性。
6. 记录运行次数、套餐限制、失败日志和通知方式。
7. 完成 demo 后，再写报价、交付边界和维护说明。

## 可复制的交付边界

```text
This automation will be tested with sample data first.
The delivery includes the workflow setup, test record, known limitations, and handover notes.
Live account access, sensitive data, billing settings, or production changes require separate approval before implementation.
```

中文版本：

```text
本自动化会先使用样例数据测试。
交付内容包括流程配置、测试记录、已知限制和移交说明。
真实账号授权、敏感数据、计费设置或生产环境变更，需要单独确认后再执行。
```

## 明日待办

- 登录 Make，核对应用连接、运行次数、日志和错误处理界面。
- 登录 Zapier，核对应用连接、任务计费、Zap 历史记录和权限提示。
- 准备一份不含客户真实信息的样例数据。
- 检查客户是否需要团队空间、移交文档或访问撤销流程。
- 人工复核是否涉及敏感数据、付款动作、账号权限或对外消息发送。

## 风险提醒

不要把自动化接到真实客户数据后再测试。不要让流程在未经确认的情况下修改、删除、发送或创建业务记录。不要承诺工具连接长期不会失效。任何自动化都需要日志、失败通知和人工兜底。

如果客户希望你连接他的邮箱、CRM、支付工具、云盘或团队聊天工具，先确认权限范围和测试方式。新手可以做 demo 和低风险小流程，但不能把复杂生产系统当作练习场。
