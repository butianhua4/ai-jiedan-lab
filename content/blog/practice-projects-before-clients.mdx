# 没有客户时怎么做练习项目

没有客户时，不代表只能等。你可以先做练习项目，把项目需要的基础能力拆开练：理解需求、拆范围、做小改动、运行检查、保存截图、写交付说明。关键是要诚实标注这是练习项目，不要把它包装成真实客户案例。

练习项目的目标不是做一个很大的产品，而是留下可信证据。一个小到能完成的页面修复、报错排查或模板整理，比一个空泛的“大型平台练习”更适合新手。

## 适合谁

适合还没有客户、没有订单、没有平台评价，但已经愿意动手做作品集的新手。你可以用自己的项目、公开练习项目或模拟需求来训练流程。

也适合正在使用 Codex、Cursor、ChatGPT 或 Claude Code 辅助学习的人。AI 可以帮你生成初稿，但你要负责判断、验证和记录。

## 不适合谁

不适合想伪造客户项目、伪造收入、伪造评价或复制别人案例的人。练习项目可以公开，但来源必须真实。

也不适合一开始就挑战高风险项目，例如支付、生产数据库、账号安全、隐私数据、复杂后端或真实客户服务器。练习阶段先做低风险任务。

## 风险提醒

不要把模拟需求写成真实客户需求。可以明确写“个人练习项目”“模拟客户需求”“本地复现案例”。诚实标注不会降低价值，反而让作品集更可信。

不要使用未经授权的商业素材、客户截图、付费模板或他人代码。需要授权、注册、绑定账号或开通服务的事项，先写成明日待办，不要为了练习硬做。

## 具体步骤

第一步，选一个小主题。比如修复移动端卡片溢出、写一个报价单模板、整理 npm 报错排查、做一个简易作品集页面。

第二步，写模拟需求。用 3-5 句话说明背景、目标、限制和验收方式。不要只写“做一个好看页面”。

第三步，保存改动前证据。截图、报错、命令输出、原始文案都可以。没有证据，后面很难写案例。

第四步，小步完成。每次只解决一个问题，不做无关重构。使用 AI 工具时，记录它帮了哪一步，你人工复核了哪一步。

第五步，运行检查。网页项目可以跑 lint、build 或手动浏览器检查；文档项目可以检查链接、错别字和事实边界。

第六步，写复盘。复盘包含做了什么、怎么验证、还没做什么、下次怎么改。

## 适合新手的练习项目

低风险项目可以从这些开始：

- 修复一个移动端布局问题
- 做一个报价单模板页面
- 写一篇报错排查记录
- 整理英文客户回复模板
- 做一个个人作品集首页
- 把旧草稿改成可审核文章
- 做一个本地工具使用记录

这些项目都能控制范围，也容易保存证据。

## 不建议一开始做的项目

暂时避开支付系统、登录权限、生产数据库、真实客户后台、自动化批量消息、复杂爬虫、公司内部工具和高风险安全修复。它们可能涉及法律、隐私、平台规则和客户授权。

如果你真的想学习，可以只写学习笔记，不要把它放进公开作品集当交付能力证明。

## 模拟需求怎么写

可以这样写：

```text
项目类型：个人练习项目
目标：修复一个移动端价格卡片按钮溢出问题
限制：不改后端，不接入支付，不部署生产环境
验收：375px 宽度下按钮文字不超出卡片，并保存前后截图
```

这段模拟需求很小，但能训练真实项目里的范围意识。

## 作品集怎么呈现

作品集里要写清楚：这是练习项目，问题是什么，改了什么，验证方式是什么，限制是什么。不要只放一张漂亮截图。

可以写“AI 辅助生成初稿，人工检查 diff 和构建结果”。这样既说明工具使用，也说明你没有把 AI 输出直接当最终交付。

## 每周怎么安排

一周只做 1-2 个练习项目就够了。一个技术练习，一个沟通或模板练习。完成后写复盘，而不是不断开新坑。

练习项目做完后，放进 [自由职业每周复盘模板](/blog/freelance-weekly-review-template)，记录证据和下一步。时间久了，作品集会自然积累出来。
