# Vercel 部署成功但页面 404：新手排查顺序

Vercel 显示 deployment ready，并不代表所有页面都能正常访问。部署成功只说明构建和发布流程走完了；页面 404 通常说明路由、输出目录、项目根目录、动态路径、域名或框架配置还有问题。新手最容易误判的一点是：把“部署成功”当成“网站已验收”。

这篇草稿专门承接“Vercel 部署成功但 404”“部署后页面打不开”“preview 正常域名不正常”等搜索需求。构建失败的排查可以先看 [Vercel build failed 常见原因](/blog/vercel-build-failed-causes)。

## 适合谁

适合已经看到 Vercel 部署成功，但打开首页、详情页、后台页或自定义域名时出现 404 的新手。你可能不确定问题出在代码、路由还是平台设置，这篇会先帮你按顺序分层。

也适合做上线验收的人。客户通常不关心 Vercel 后台是不是绿色，而是关心他要访问的页面是否能打开、链接是否正确、移动端是否可用、域名是否生效。

## 不适合谁

不适合还没有构建成功的人。如果 build failed，先不要排查 404，而要回到构建日志。否则你会把两个不同阶段的问题混在一起。

如果项目涉及生产域名、用户登录、数据库、支付或后台权限，新手不要独立修改高风险配置。你可以做路径检查和预览验证，但生产切换要让客户确认。

## 原因一：首页路径和实际路由不一致

很多 404 来自访问路径错误。比如项目只有 `/app`、`/dashboard` 或 `/landing`，但你打开根路径 `/`；或者 Next.js、React Router、Astro 的路由文件和你以为的页面地址不一致。

排查时先列出项目应该有的页面：主页、列表页、详情页、登录页、后台页、静态文件。每个页面都用 Vercel 预览链接测试一遍，不要只看首页。如果只有某些路径 404，说明问题更可能在路由或动态页面。

## 原因二：输出目录或框架配置不匹配

有些项目构建成功，但输出目录配置错了，Vercel 发布的不是你想要的文件。静态项目常见输出目录包括 `dist`、`build`、`out`，不同框架不一样。配置错时，构建看似通过，访问时却找不到页面。

新手不要凭记忆填写 Output Directory。应该看框架文档、项目配置和构建日志中生成的目录。改完后重新部署，再访问预览链接验证。

## 原因三：Root Directory 选错

如果仓库里有多个项目，Vercel 可能从错误目录部署。它可能部署了一个空目录、旧项目，或者只构建了 monorepo 里的另一个应用。此时页面 404 并不奇怪。

你要检查仓库结构，确认真正的网站目录在哪里，再看 Vercel Project Settings 里的 Root Directory。这个问题在 `apps/web`、`packages/site`、`frontend` 这类结构里很常见。

## 原因四：动态路由没有生成或接口不可用

详情页、文章页、产品页这类动态路径更容易出问题。页面可能依赖 CMS、数据库、API 或构建时数据。如果构建时没有拿到数据，或者动态路径没有被正确生成，就可能只在特定路径 404。

排查动态路由时，要记录一个正常路径和一个失败路径，对比数据来源、slug、大小写、尾部斜杠和接口响应。不要只说“详情页坏了”，要给出具体 URL。

## 原因五：域名和预览链接表现不同

如果 Vercel 预览链接正常，自定义域名 404，要重点看域名绑定、DNS、项目关联和缓存。域名问题需要时间生效，也需要确认当前域名指向的是不是这个 Vercel 项目。

不要在没有确认前频繁切换 DNS。先记录当前解析、Vercel 域名状态、预览链接结果和自定义域名结果，再决定下一步。

## 风险提醒

404 排查看似简单，但牵涉生产域名时就不简单。错误修改 Root Directory、Output Directory 或域名配置，可能让原本可访问的页面失效。每次修改前都要记录旧值，并确认能回退。

如果客户要求你直接操作主域名、删除项目、替换生产部署或处理账号权限，先暂停确认范围。新手可以做诊断和建议，不要把高风险操作当作普通检查。

## 具体步骤

第一步，确认构建是否真正成功。如果仍有 build failed，先处理构建问题。

第二步，用 Vercel 预览链接测试根路径、关键路由、动态路径和静态资源。

第三步，检查 Root Directory、Framework Preset、Build Command 和 Output Directory。

第四步，对比预览链接和自定义域名。如果预览正常域名异常，重点查域名绑定和 DNS。

第五步，把访问结果整理成验收表：URL、状态、截图、可能原因、下一步。需要模板可以从 [工具导航](/tools) 下载，或联系人工协助整理上线验收报告。
